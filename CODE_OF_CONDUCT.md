# Code of Conduct

## Our Goal

We want this project to be useful, welcoming, and practical for people building real Unity workflows.

## Expected Behavior

Participants in this project are expected to:

- be respectful and constructive
- assume good intent while still giving honest technical feedback
- focus on the code, workflow, or idea instead of the person
- help newcomers get unstuck without gatekeeping
- keep disagreements specific and solution-oriented

## Unacceptable Behavior

The following are not acceptable in project spaces:

- harassment or personal attacks
- hateful or discriminatory language
- intimidation, threats, or doxxing
- bad-faith disruption, trolling, or repeated hostility
- sharing private information without permission

## Enforcement

Project maintainers may remove comments, close discussions, reject contributions, or block participants whose behavior harms the project or its contributors.

## Reporting

If something happens that should be reviewed privately, report it through the contact path described in `SECURITY.md` or the repository contact methods.
